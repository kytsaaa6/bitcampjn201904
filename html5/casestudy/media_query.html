git push 는 모든 지역 브랜치를 git push 하지 않습니다 : 어떤 원격 브랜치를 푸시해야하는지 어떻게 알 수 있습니까? 특정 원격 브랜치로 푸시하도록 구성된 로컬 브랜치 만 푸시합니다. Git (1.6.5.3)의 제 버전에서 git remote show origin 실행하면 실제로 어떤 브랜치가 푸시를 위해 구성되어 있는지 인쇄합니다. Local refs configured for 'git push': master pushes to master (up to date) quux pushes to quux (fast forwardable) Q.하지만이 모든 것에 대해 걱정하지 않고 master 할 수 있습니다! git clone 할 때, 기본적으로 로컬 master 브랜치를 설정하여 원격의 master 브랜치 (로컬에서는 origin/master 라고 함)에 푸시합니다. 따라서 master 에서만 커밋하면 간단한 git push 는 항상 변경 사항을 다시 푸시합니다. . 그러나 게시 한 출력 스 니펫 (snippet)에서 개발자는 development라는 브랜치에 있습니다.이 브랜치는 아무 것도하지 않을 것이라고 추측합니다. 인수없이 git push 하면 해당 분기에서 커밋을 수행하지 않습니다. "모든 최신 정보"라고 말하면 "푸시하는 방법을 알려준 모든 지점"이 최신 정보라는 의미입니다.
